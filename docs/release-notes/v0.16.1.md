# javalens-manager v0.16.1 — Windows runtime fixes, Claude Desktop target, scroll perf (smoke patch)

Patch release closing the findings from the v0.16.0 live smoke (Linux + first Windows run).
Runtime unchanged — any fork v1.8.5+ works; pairs with fork v1.9.0.

## Windows

- **No more console window (#16).** The resident JVM (and the Test-Services probe) spawned
  `java.exe` without `CREATE_NO_WINDOW`, so every workspace start popped a console on
  Windows — and it lingered after stop, blocking the port on restart. Both spawn sites now
  set the flag; the JVM runs windowless. (No-op on Linux/macOS.)
- **Claude Code finds the service (#18).** Path detection listed a bogus `~/.claude/mcp.json`
  first; on a fresh box the entry was written where Claude Code never reads. `~/.claude.json`
  (= `%USERPROFILE%\.claude.json`) now leads.
- **Claude Desktop is now a deploy target (#17).** Previously the manager only targeted
  Claude Code. Claude Desktop is a distinct client — detected via the OS config dir
  (`%APPDATA%\Claude\claude_desktop_config.json` on Windows,
  `~/Library/Application Support/Claude/…` on macOS), deployed with the native-HTTP
  `{type:"http", url, headers}` shape, and shown as its own row in Settings → MCP Config
  Locations and the deploy picker.

## Linux

- **Faster scrolling (#20).** WebKitGTK (the Linux webview) recomposites the whole backdrop
  every scroll frame when a sticky element overlaps the scroll region. Two sticky elements
  (the page header and the bulk-action bar) carried a `backdrop-filter: blur(2px)` that was
  invisible at their opacity; both are now opaque. Added `contain: paint` to the main and
  nested scroll containers so inner *and* outer (whole-page) scrolling isolate their
  repaints. (Windows/WebView2 was already smooth; untouched.)

## UX & docs

- **Autoscan Discover button (#19).** With Recursive-search on, Browse already auto-scans —
  so the now-redundant "Discover" button hides once results are showing for the current
  path. It reappears for a hand-typed or edited path.
- **README timeline order (#15).** `v0.14.1` was listed after `v0.16.0`; moved to its slot.
- **Help page (#21).** Installation section now covers Linux / macOS / Windows (was
  Linux-only). Embedded UI screenshots still predate the v0.16.0 layout and are tracked for
  a follow-up recapture.

## Fixed

Bugs [#15–#21](../bugs.md). All fixes built green on the 5-platform CI matrix. Two
Windows-only behaviours can only be confirmed on real hardware: #16's restart/port-bind
(the console-window suppression is certain; if a port still fails to rebind after stop, the
follow-up is a Windows Job Object to kill the full process tree) and #17's Claude Desktop
url-based entry (if a given Desktop build won't read it, the follow-up is an `mcp-remote`
stdio-shim entry shape).
