# javalens-manager v0.14.0 — Sprint 14 closeout (hardening + Reload-all + Autostart + macOS packaging)

Feature + hardening release on top of v0.13.1. Closes three open manager bugs, adds two requested tray + dashboard features, opens the manager to Mac users via a new CI matrix entry. No fork release paired — fork bugs #6 / #7 stay OPEN in `javalens-mcp/docs/bugs.md` and roll into whichever future fork release ships them.

## What changed

### Bug fixes

- **bugs.md #3 — single-instance enforcement.** Wired `tauri-plugin-single-instance` v2.4.2 as the first plugin in the Tauri Builder chain. Double-launching from the dock / app menu now raises the existing window instead of spawning a second process + second tray icon. Standard cross-platform IPC (Unix domain socket on Linux/macOS, named pipe on Windows). Callback does `unminimize → show → set_focus` on the running process's `main` window.

- **bugs.md #2 — process-death → Failed.** External `kill -9` (or a crash / OOM) of a managed JVM now flips the workspace to `RuntimePhase::Failed` instead of `Stopped`. Tray glyph goes red `✗` (was gray `○`, indistinguishable from a user-initiated *Stop*). `RuntimeStatusRecord` gains an additive `exitCode: number | null` field; `RuntimePhase` stays a 4-variant unit enum to preserve the frontend's `status.phase === "failed"` string-match wire format. `try_join_running_workspace*` was the bug site — all stop paths (`stop_runtime`, `stop_workspace_runtime`, `remove_project_runtime`) `handles.remove()` + `kill()` + `wait()` synchronously, so reaching the `try_wait → Some` branch with the handle still present IS the external-death case.

- **bugs.md #1 — webview blank on aarch64 (full fix).** `WEBKIT_DISABLE_DMABUF_RENDERER=1` is now baked into the AppImage's `AppRun` AND the `.deb`'s `/usr/bin` wrapper at CI build time. v0.13.1's `install.sh` curl one-liner covered itself; v0.14.0 closes the gap for users who install via `dpkg -i` or double-click the AppImage directly. Every install path is covered. `install.sh` simplified to download the AppImage directly to `$BIN_DIR/$APP_NAME` (drops the manual-wrapper-write path).

### New features

- **Reload all (tray + dashboard).** Stop every workspace, poll until each phase reaches `Stopped | Failed` (30 s deadline), then start them all. Sequential rather than fire-and-forget so the respawn doesn't race a workspace that's still mid-shutdown (where `start_runtime` would otherwise fast-path into "already running"). Tray menu entry "Reload all services" between Start all and Stop all; dashboard toolbar button between Stop all and Delete all.

- **Autostart on boot.** Manager auto-launches at session login via `tauri-plugin-autostart`. Two UI entry points, one backend setting:
  - Settings → Machine Runtime Controls → "Autostart on boot" checkbox.
  - Tray menu → "Autostart on boot" checkable (between Stop all and Quit).

  Per-OS plumbing handled by the plugin: Linux writes `~/.config/autostart/*.desktop`, macOS registers a LaunchAgent, Windows touches the registry Run key. Default is opt-in (`false`) — power users keep control of what's tray-resident at login. The OS-level autostart is reconciled with the saved setting at every launch in the Tauri `setup` block (best-effort; errors don't block startup).

### Packaging

- **macOS builds in CI.** `release.yml` matrix gains `macos-14` (Apple Silicon) and `macos-13` (Intel). Both free on public-repo tier. `tauri.conf.json` bundle targets switched to `"all"` so each runner picks the targets appropriate for its host (`deb` + `appimage` on Linux, `dmg` + `app` on macOS). The existing apt-install guard auto-excludes macOS runners. **Unsigned in v0.14.0** — Apple Developer signing is a separate later track. README now documents the Gatekeeper bypass (`xattr -d com.apple.quarantine /Applications/javalens-manager.app` or right-click → Open the first time).

- **install.sh — Darwin branch.** The same `curl … | bash` one-liner now works on macOS too. New `install_macos()` function: arch-detects (`x86_64` / `arm64`), curls the matching `.dmg`, mounts it via `hdiutil attach -nobrowse -quiet -mountpoint`, copies the `.app` to `/Applications/`, unmounts, strips the quarantine xattr. EXIT trap cleans up on failure. Linux flow below the dispatcher unchanged.

- **CI restructure (Linux env-var bake-in).** `release.yml` now uses `tauri-action` as build-only (no upload), patches the Linux artifacts in a Linux-only post-bundle step (`--appimage-extract`, sed-inject `WEBKIT_DISABLE_DMABUF_RENDERER=1` before `^exec ` in `AppRun`, `appimagetool --appimage-extract-and-run` to repack; `dpkg-deb -R`, move binary to `/usr/lib/`, write a 5-line wrapper at `/usr/bin/`, `dpkg-deb -b` to repack), then uploads via `softprops/action-gh-release@v2`. Same draft-release behavior as before; cleaner data flow.

## Compatibility

- Drop-in upgrade from v0.13.1. No data-format changes; existing `projects.json` and `settings.json` work as-is. The new `autostart_on_boot` settings field is `#[serde(default = "default_autostart_on_boot")]` (defaults to `false`) so pre-v0.14.0 `settings.json` files load cleanly. The new `exit_code` field on `RuntimeStatusRecord` is `#[serde(default)]` for pre-v0.14.0 `runtime-state.json`.
- Frontend wire format for `RuntimePhase` is unchanged (`"failed"` etc as strings, not struct variants). The optional new `exitCode` field on `RuntimeStatusRecord` doesn't break existing frontend checks.
- Mac users: see README → Installation → macOS. Unsigned, so Gatekeeper bypass required.
- Linux users who had the v0.13.1 install.sh wrapper at `$BIN_DIR/$APP_NAME`: re-running v0.14.0 install.sh overwrites it with the AppImage directly (the wrapper is no longer needed — the AppImage bakes the env-var). Legacy `$BIN_DIR/$APP_NAME.AppImage` is auto-removed.
