# javalens-manager v0.16.0 — Windows, autoscan import, per-client deploy schemas, lifecycle leak closure (Sprint 16)

Pairs with fork **v1.9.0** (already Latest; any v1.8.5+ runtime works — the manager
remains a pure orchestrator).

## Windows joins the platform matrix

First Windows release: **x64 and ARM64**, each as both a WiX `.msi` and an NSIS
`-setup.exe`. Unsigned for now — a one-time SmartScreen **More info → Run anyway** is
required (code-signing is a separate later track; see the README's Windows section).
WebView2 installs automatically where missing. The platform matrix is now Linux
amd64/arm64 + macOS Apple Silicon + Windows x64/ARM64; macOS Intel remains explicitly
out of scope.

## Recursive search (autoscan) in Add Project

The Register Project form gains a **Recursive search (autoscan)** checkbox: tick it,
Browse to a parent folder (e.g. `~/Projects`), and the manager scans recursively
(depth ≤ 6) for Maven/Gradle and Eclipse/PDE projects — results unfold inline into the
familiar candidate checkbox-list; prune and **Import selected** into the active
workspace. Hand-typed paths use the **Discover** button. No `.code-workspace` seed file
needed anymore — the gap surfaced in the Sprint 14 live smoke is closed. Both import
flows re-discover server-side; client-supplied paths are never trusted directly.

## Per-client MCP deploy schemas (bug #10)

Deploy now writes the entry shape each client's parser accepts:

| Client | Entry shape |
|---|---|
| Antigravity | `{ "serverUrl": "http://127.0.0.1:<port>/mcp", "headers": { … } }` — no `type` |
| Cursor / Claude Code / IntelliJ | `{ "type": "http", "url": …, "headers": { … } }` (v0.15.1 shape, unchanged) |

Antigravity previously rejected the generic shape with "serverURL or command must be
specified" and needed a hand-patched config that every re-deploy reverted. The schema
table is centralised in one function; the post-write validator checks per-client.

## Lifecycle leak closure (bugs #11, #12, #13)

- **Rename** migrates the workspace's `(port, token)` entry — deployed endpoints stay
  valid across renames instead of silently pointing at a dead port.
- **Delete** (workspace, or last project of one) stops the resident and releases the
  port back to the allocator pool.
- **Quit** now stops resident JVMs on every quit path (best-effort, never blocks exit).
- **One-shot orphan prune:** on load, `workspaces[]` entries naming no current workspace
  are dropped — with a `projects.json.bak-<timestamp>` backup first and a log line per
  pruned entry. Existing installs reclaim the ports leaked by pre-v0.16.0 renames and
  deletions on first launch.

## Deploy correctness (bug #14)

- Workspace **add / rename / delete auto-refresh** the configs of clients you have
  already deployed to (detected by the presence of `jl-`/`javalens-` managed entries;
  never-deployed clients are left untouched). A freshly added workspace is visible to
  your MCP clients without a manual Deploy click.
- `build_deploy_servers` no longer silently drops workspaces that fail runtime
  resolution — the errors ride on every written client result and the deploy summary
  states how many workspaces were omitted.
